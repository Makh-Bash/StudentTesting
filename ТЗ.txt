1. Общие сведения
    1.1 Наименование системы: Система тестирования студентов онлайн тренинг центра "Student Testing"
        Условное обозначение: "ST"
    1.2 Наименование предприятия заказчика: "FIiIT_IT"
        Наименование предприятия разработчика: "DagIT"
    1.3 Компания "FIiIT_IT" издала указ №142 по созданию нового портала для онлайн тестирования
        обучающихся, целью которого является автоматизация тестировочного процесса
    1.4 Плановые сроки начала: 14.10.2022
        Плановые сроки конца: 28.10.2022

2. Назначение и цели системы: автоматизация тестирования студентов, оптимизация учебного процесса
   Цель: реализовать модель дистанционного тестирования в учебном процессе для достижения положительного
   технологического эффекта
3. Характеристика объектов автоматизации: Образовательное учреждение ДГУ,
   Факультет Информатики и информационных технологий, система тестирования обучающихся

   Возможности студентов: регистрироваться, изменять профиль, проходить тесты, узнавать результаты, связь с другими
   пользователями.
   Возможности преподавателей: возможности студентов + создание, изменение и удаление своих тестов.
   Возможности администратора: возможности преподавателей + просмотр, блокировка и удаление пользователей системы

4. Требования к системе
    4.1 Архитектура: Небольшой корпоративный сайт
        Структура:
            1. Страница входа/регистрации
            2. Приветствие
            3. Каталог тестов (Поиск тестов)
                3.1 Выполнение тестов
                3.2 Получение результатов
            4. Создание тестов
            5. Профиль пользователя
                5.1 Редактирование профиля
            6. Кабинет администратора
                6.1 Список пользователей (Удаление, блокировка)
                6.2 Список тестов (Удаление)
            7. Информация о разработчиках (Контакты)
        Требование к дизайну: отсутствует

    4.1.1 Структура БД
        Пользователи:
            ID,
            Имя,
            Фамилия,
            Курс,
            Специальность (Студенты),
            Логин,
            Почта,
            Пароль,
            Дата рождения,
            Время регистрации,
            Роль (Студент, Преподаватель)
        Тесты,
        Решения.
    4.2 См. файл
    4.3 Необходимость взаимодействия с другими системами: отсутствует
    4.4 Требования к надежности: шифрование паролей, уникальные логины/почты пользователей,
        двухэтапная аутентификация
    4.5 Требования к администраторам: соблюдение правил, соблюдение анонимности данных пользователей,
        своевременная поддержка пользователей
    4.6 Требования к режиму функционирования:
        Штатный режим: без перерывов.
        Аварийный режим: сообщение об ошибке
5. Требов   ание к функциям системы:
    Функция "Тесты": создание, редактирование, выбор тестов, прохождение, получение результатов, удаление тестов.
    Функция "Профиль": регистрация, вход, редактирование, выход с аккаунта.
